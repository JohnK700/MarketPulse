## About The Project

MarketPulse is a full-stack sales analytics platform built with Python Flask. Users upload CSV sales data and receive automated statistical summaries, trend visualizations, and a 30-day ML forecast — all through a personalized dashboard.

**Key concepts demonstrated:**
- RESTful backend with session-based user authentication and role-based access control
- Automated data pipeline for CSV ingestion, cleaning, and transformation using Pandas
- Time-series forecasting with Facebook Prophet including upper/lower confidence intervals
- Dynamic chart generation (4 chart types) using Matplotlib and Seaborn
- MVC architecture with clear separation of controllers, models, and views
- Automated test suite with Pytest

## Getting Started

### Prerequisites

- Python 3.9+
- pip

### Installation

1. Clone the repo
   ```sh
   git clone https://github.com/JohnK700/MarketPulse.git
   ```
2. Navigate into the project directory
   ```sh
   cd MarketPulse
   ```
3. Install dependencies
   ```sh
   pip install -r requirements.txt
   ```
4. Run the application
   ```sh
   python run.py
   ```
5. Open your browser and go to `http://127.0.0.1:5000`

## Usage

1. **Register / Log in** — create an account or sign in to access your personal dashboard
2. **Upload CSV** — upload a sales data file; the pipeline automatically validates, cleans, and processes it
3. **View Analytics** — explore automated daily and weekly sales summaries with key statistics (total, average, max, min)
4. **View Forecast** — get a 30-day sales forecast with confidence intervals generated by Facebook Prophet
5. **View Charts** — view 4 auto-generated charts saved to your personal storage

### Expected CSV Format

Your CSV should include the following columns:

```
,Food,Drink,Modifier,Gross,Tax,Total
08/31,"$4,549.60",$232.85,$558.05,"$5,343.50",$423.92,"$5,601.79"
09/01,"$4,711.60",$268.94,$488.30,"$5,468.84",$432.88,"$5,712.63"
...
```

Dollar signs and commas in values are handled automatically by the data pipeline.

## Implementation Details

| Component | Details |
|---|---|
| **Authentication** | Session-based login with account CRUD operations and role-based access control; user data stored in `users.json` |
| **Data pipeline** | CSVs are validated for required columns (`Food`, `Drink`, `Modifier`, `Gross`, `Tax`, `Total`), stripped of currency formatting, and parsed into typed DataFrames via Pandas |
| **Forecasting** | `SalesForecaster` class fits a Facebook Prophet model with daily and weekly seasonality, then generates a 30-day prediction with `yhat`, `yhat_lower`, and `yhat_upper` columns |
| **Visualization** | Matplotlib and Seaborn generate 4 chart types: daily trend (line), weekly trend (line), forecast with confidence interval shading, and daily sales distribution (histogram + KDE); saved as user-specific PNGs |
| **Architecture** | MVC pattern — `controllers/` handles routing and business logic, `models/` manages user data, `templates/` renders HTML views |
| **Testing** | Pytest suite in `tests/` covers login flow (`test_login.py`) and CSV input processing (`test_input_controller.py`) |

## Project Structure

```
MarketPulse/
├── run.py                        # App entry point
├── requirements.txt
├── src/
│   ├── app.py                    # Flask app and route definitions
│   ├── controllers/
│   │   ├── input_controller.py   # SalesForecaster — Prophet ML pipeline
│   │   ├── sales_data_controller.py  # CSV upload, graph generation
│   │   ├── account_controller.py
│   │   └── registration_controller.py
│   ├── models/
│   │   └── accounts.py           # User account model
│   ├── templates/                # HTML views (login, dashboard, upload, etc.)
│   └── data/
│       ├── uploads/              # User-uploaded CSVs (per-user folders)
│       └── graphs/               # Generated chart PNGs (per-user folders)
└── tests/
    ├── test_login.py
    └── test_input_controller.py
```
